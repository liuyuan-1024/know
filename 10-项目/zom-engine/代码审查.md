【项目背景】
本项目是一个纯粹的单机底层文本编辑引擎底座。它只提供内存级别的纯文本编辑能力，为上层多语言、多平台的编辑器宿主提供服务。
**引擎的绝对边界：** 没有 UI、没有渲染、没有操作系统 I/O（文件读写）、没有系统剪贴板操作、不处理平台键盘/鼠标原生事件。所有外部依赖必须通过抽象 trait 注入。

1. 🚫 **Engine-only 绝对边界检查**：
   - 严查引擎内部是否掺杂了任何 UI 渲染逻辑或宿主职责（如 `std::fs`、系统原生事件等）。
   - 确保引擎代码完全处于“无头（Headless）”状态，对外部环境“零感知”。

2. 🏗️ **领域防腐与坐标系绝对论**：
   - 严防“语义倒置”：引擎的接口、结构必须基于“文本编辑领域语言”，绝不允许包含宿主概念（如 Window, LSP）。
   - 坐标系唯一真理：引擎内部极深核心必须且只能使用“绝对字节偏移量（usize）”。行/列坐标等必须作为边界处的“投影/视图”隔离。

3. ⚙️ **纯粹状态机与 CQRS / Delta 驱动**：
   - 确定性与纯函数：相同的初始状态 + 相同的 Command 序列，必须推演出绝对一致的最终状态。
   - Delta/Patch 驱动：任何修改动作绝不能是“默默突变内存”，必须产出严密的 `Delta` 变更集。严禁滥用 `Rc<RefCell<T>>` 或 `Arc<Mutex<T>>` 逃避所有权纪律。

4. 🚀 **极端性能与大文件内存纪律**：
   - 面对 GB 级大文件，严查 O(N) 级别的全量遍历或拷贝。强制要求 $O(\log N)$ 的渐进式计算（如 Rope/Piece Table）和底层数据结构的碎片整理（Compaction）。
   - 严厉打击无谓的 `String::clone`，全链路审查 Zero-copy 的贯彻情况。

5. 🛡️ **病态边缘场景与绝对健壮性**：
   - 绝对零容忍任何 `unwrap()` 或 `panic!()` 逃逸到外层。必须区分内部致命错误（断言）与可恢复的外部编辑错误。
   - 严禁生硬操作 `byte` 或 `char` 而无视 Unicode 字形簇（Grapheme）边界的行为；必须能抗住无换行的数百万字符或混乱换行符等病态输入。

6. 🧪 **确定性验证与 Fuzzing 测试友好度**：
   - 检查代码是否高度解耦以支持无宿主环境的脱机独立测试。

8. 💣 **毫不妥协的重构与历史包袱清理**：
