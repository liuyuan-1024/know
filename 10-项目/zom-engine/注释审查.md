【项目背景与架构底线】
本项目是一个纯粹的单机底层文本编辑引擎底座，为上层宿主提供内存级别的无头（Headless）纯文本编辑服务。
- **绝对边界：** 无 UI、无渲染、无 I/O、无原生事件。只有纯粹的数据流。
- **架构核心：** 绝对字节偏移量坐标系（usize）、CQRS/Delta 驱动、零拷贝极速状态机。

【三维注释生成策略 (核心纪律)】

1. 🗺️ **模块级 (`//!`) - 心智模型、图解与闭环示例**：
   - **建立模型**：阐述该模块在 CQRS 或状态机中的角色。
   - **视觉辅助**：对于复杂数据结构（如文本树、坐标映射），**必须使用清晰的 ASCII Art 字符图**辅助解释内存或状态变换。
   - **最小闭环 Doctest**：必须包含一个模块级的“最小使用场景 Doctest”，展示如何构造初始状态机 -> 派发 Command -> 断言产出的 Delta/状态。
   - **边界隔离**：明确指引宿主：“寻找 UI 绑定的请去 `examples/`，本模块只处理纯粹的机器契约”。

2. ⚖️ **公开 API (`///`) - 极度严苛的契约驱动 (DbC)**：
   - **拒绝废话**：严禁简单重复函数签名。必须一句话点明业务本质。
   - **坐标系与 Panic 契约**：凡涉及 `usize` 绝对字节偏移量的 API，必须强制在 `# Panics` 中警告：“当偏移量不落在合法的 UTF-8 字符边界（或 Grapheme Cluster 边界）时将触发 Panic”。
   - **Error 契约**：必须包含 `# Errors` 章节说明业务可恢复错误。
   - **Safety 契约**：一旦涉及 `unsafe` 函数或块，强制且必须提供详细的 `# Safety` 章节，极其严苛地说明调用者必须保证的内存/生命周期不变式。
   - **Doctest 契约**：`# Examples` 必须是极度干练的脱机状态机推演。绝不包含伪造的 UI 逻辑。

3. 🧠 **内部实现 (`//`) - 意图与算法妥协**：
   - 包含复杂内部算法（如 $O(\log N)$ 的树重排、Zero-copy 的生命周期 trick）时，使用 `//` 明确解释 **“为什么（Why）这样写”** 以及 **“性能/内存的考量点”**。不要解释“代码在干嘛”。

4. 🚫 **词汇禁区**：
   - 严禁出现 "Draw", "Render", "Window", "Click" 等宿主端词汇。使用 "Host", "Apply Delta", "Measure" 等领域词汇。
