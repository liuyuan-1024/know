# 开发前准备
**nginx：**
```bash
# linux
# 启动
sudo systemctl start nginx
# 关闭
sudo systemctl stop nginx
# 查看
sudo systemctl status nginx

# mac
# 启动
brew services start nginx
# 重启动
brew services restart nginx
# 关闭
brew services stop nginx
# 检查状态
brew services list | grep nginx
# 重载配置（修改配置，无需重启）
sudo nginx -s reload
```

**wireguard：**
```bash
# linux
# 启动
sudo wg-quick up zmn
# 关闭
sudo wg-quick down zmn
# 查看
sudo wg
wg show zmn

# mac直接启动客户端
```

# 流程
1. 拉取远程 main 分支的最新代码
2. 修改专属分支，完成任务
3. 提交代码前，暂存修改，再次拉取远程 main 分支，保证提交前是最新的代码
4. 恢复暂存，解决冲突，提交并推送代码
5. github 提交 PR，确认合并。

```bash
# 或者直接在专属分支上：拉取远程 main 并合并到当前分支
git pull origin main

# 如果已经修改了专属分支，有需要合并远程main分支，那么就暂存当前的修改
git stash push -m "暂存修改的相关提示"
# 恢复最近一次暂存，并删除该暂存记录：
git stash pop
```
