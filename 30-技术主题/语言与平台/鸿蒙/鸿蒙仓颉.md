## 工程管理
### APP包结构
<font style="color:rgba(0, 0, 0, 0.9);">应用/服务发布形态为 </font>**<font style="color:rgba(0, 0, 0, 0.9);">APP Pack</font>**<font style="color:rgba(0, 0, 0, 0.9);">（Application Package），它是由一个或多个</font>[<font style="color:rgb(10, 89, 247);">HAP（Harmony Ability Package）</font>](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/hap-package-V5)<font style="color:rgba(0, 0, 0, 0.9);">包以及描述 APP Pack 属性的 pack.info 文件组成，每个 HAP 中可以包含不同的内容。</font>

<font style="color:rgba(0, 0, 0, 0.9);">一个 HAP 在工程目录中对应一个 Module，它由代码、资源、三方库及应用/服务配置文件组成。HAP 可以分为 Entry 和 Feature 两种类型。</font>

+ **<font style="color:rgb(36, 39, 40);">Entry</font>**<font style="color:rgb(36, 39, 40);">：应用的主模块，作为应用的入口，提供了应用的基础功能。</font>
+ **<font style="color:rgb(36, 39, 40);">Feature</font>**<font style="color:rgb(36, 39, 40);">：应用的动态特性模块，作为应用能力的扩展，可以根据开发者的需求和设备类型进行选择性安装。</font>

<font style="color:rgba(0, 0, 0, 0.9);">仓颉仅支持 Stage 模型开发应用，Stage 模型应用程序包结构如下图所示。</font>

+ <font style="color:rgb(36, 39, 40);">ets 目录用于存放应用代码编译后的字节码文件，仅在仓颉 + ArkTS （+ C++）的混合工程中存在该目录。</font>
+ <font style="color:rgb(36, 39, 40);">libs 目录用于存放库文件。库文件是应用依赖的第三方代码（.so 二进制文件）。</font>
+ <font style="color:rgb(36, 39, 40);">resources 目录用于存放应用的资源文件（字符串、图片等），便于开发者使用和维护。</font>
+ <font style="color:rgb(36, 39, 40);">resources.index 是资源索引表，由IDE编译工程时生成。</font>
+ <font style="color:rgb(36, 39, 40);">module.json 是 HAP 的配置文件，内容由工程配置中的 module.json5 和 app.json5 组成，该文件是 HAP 中必不可少的文件。DevEco Studio 会自动生成一部分默认配置，开发者按需修改其中的配置。后续解析应用，获取应用信息的主要的对象，包含如下信息：</font>
    - <font style="color:rgb(36, 39, 40);">app.json5 主要包含以下内容：</font>
        * <font style="color:rgb(36, 39, 40);">应用的全局配置信息，包含应用的 Bundle 名称、开发厂商、版本号等基本信息。</font>
        * <font style="color:rgb(36, 39, 40);">特定设备类型的配置信息。</font>
    - <font style="color:rgb(36, 39, 40);">module.json5 主要包含以下内容：</font>
        * <font style="color:rgb(36, 39, 40);">Module 的基本配置信息，例如 Module 名称、类型、描述、支持的设备类型等基本信息。</font>
        * <font style="color:rgb(36, 39, 40);">应用组件信息，包含 UIAbility 组件和 ExtensionAbility 组件的描述信息。</font>
        * <font style="color:rgb(36, 39, 40);">应用运行过程中所需的权限信息。</font>
+ <font style="color:rgb(36, 39, 40);">pack.info 是 Bundle 中用于描述每个 HAP 属性的文件，例如 app 中的 bundleName 和 versionCode 信息、module 中的 name、type 和 abilities等信息，由 IDE 工具生成 Bundle 包时自动生成，作为后续运行时解析 HAP 包，获取 HAP 信息的主要的对象。</font>

## <font style="color:rgb(0, 0, 0);">模块管理</font>
<font style="color:rgba(0, 0, 0, 0.9);">Module 是应用/服务的基本功能单元，包含了源代码、资源文件、第三方库及应用/服务配置文件，每一个 Module 都可以独立进行编译和运行。一个应用/服务通常会包含一个或多个 Module，因此，可以在工程中创建多个 Module，Module 分为 Ability 和 Library 两种类型。</font>

### <font style="color:rgba(0, 0, 0, 0.9);">创建仓颉静态库模块</font>
<font style="color:rgba(0, 0, 0, 0.9);">HAR(Harmony Archive）是静态共享包，通过HAR可以实现多个模块或多个工程共享 ArkUI 组件、资源等相关代码。HAR 不同于 HAP，不能独立安装运行在设备上，只能作为应用模块的依赖项被引用。</font>
