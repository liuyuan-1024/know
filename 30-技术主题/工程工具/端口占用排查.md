## 1.查看所有端口的占用情况

```java
netstat -ano
```

## 2.查看指定端口的占用情况

```java
netstat -ano|findstr "端口号"
```

## 3.使用tasklist查看 PID 对应的进程名

```java
tasklist|findstr "PID号"
```

## 4.结束进程

```java
taskkill /f /t /im PID
```

## 5.端口显示被占用，netstat -aon | findstr 却找不到端口

> 本机启用了 Hyper-V 功能，这个功能与 [Docker](https://so.csdn.net/so/search?q=Docker&spm=1001.2101.3001.7020) 或者 虚拟机 有关，使用 `WSL2`也需要开启这个功能。Widows 默认把某些端口范围分给了 Hyper-V ，成为了保留端口，所以我们无法使用这个端口。并且由于这个端口分配在电脑重启的时会重新随机分配
>

1.  查看 Windows 默认的 tcp 动态端口范围 

> netsh int ipv4 show dynamicport tcp

2.  查看保留端口范围 

> netsh interface ipv4 show excludedportrange protocol=tcp
>

### 解决方法

> netsh int ipv4 set dynamicport tcp start=49152 num=16383
> netsh int ipv4 set dynamicport udp start=49152 num=16383
