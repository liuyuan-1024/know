## Git基本使用流程
```git
# 克隆远程仓库代码
git clone 远程仓库地址

# 开始工作前和提交代码前执行一次pull操作，以确保本地代码是最新的。
git pull 远程仓库 分支 # 使用merge方式拉取远程仓库指定分支
git pull 远程仓库 分支 --rebase # 推荐，使用rebase方式拉取远程仓库指定分支
# refusing to merge unrelated histories 拒绝合并两个无关的历史
git pull 远程仓库 分支 --allow-unrelated-histories # 强制拉取并合并

# 1.在代码托管中心创建员创建代码仓库，即远程代码仓库
# 2.初始化本地仓库并指定默认分支为main
git init -b main
# 3.添加远程仓库地址
git remote add origin 远程仓库地址
git remote -v # 查看所有远程库的别名
git remote rename origin 新远程仓库名 # 修改远程仓库名，记得更新本地分支的跟踪关系
git remote set-url origin 新URL # 修改指定远程仓库的地址
# 4.创建并切换到该分支
git branch 分支名 #创建
git checkout 分支名 #切换
git checkout -b 分支名 #创建并切换
git branch -m 旧分支名 新分支名 # 修改本地分支名
git branch -d 分支名 # 安全删除本地分支
git branch -D 分支名 # 强制删除本地分支
git push origin --delete 分支名 # 删除远程分支
git push origin --mirror # 强制覆盖远程仓库
# 5.向暂存区中添加或移除文件，“.”代表全部文件
git add 文件 # 忽略.gitignore文件配置的目录及文件
git rm --cached build/ # 移除本地对build文件夹的跟踪（不会删除本地实际文件，仅删除Git记录）
# 6.暂存区内容提交到本地仓库；正文应<100个字符，方便在GitHub以及各种Git工具中阅读
git commit -m "标签: 标题" -m "提交信息正文"
# 7.推送本地当前分支代码到远程仓库指定分支
git push 远程仓库 远程分支
git push -u 远程仓库 远程分支 # 推送并建立跟踪关系

# 将已存在本地仓库推送到新建远程仓库
git remote add origin 远程仓库地址
git push -u 远程仓库 --all # 推送本地仓库所有分支到远程仓库并建立跟踪关系。
git push -u 远程仓库 --tags # 推送本地仓库所有标签到远程仓库，一般用不到
```

```git
# 以“dev into main”为例，main为目标分支，dev为待合并分支
git checkout main # 切换到目标分支
git pull 远程 main # 拉取最新目标分支
git merge dev # 将待合并分支合并到当前分支
# 发生冲突，解决冲突后执行下列操作
git add <冲突文件>
git commit -m "merge dev into main"
# 无冲突，直接push
git push 远程 main
```

```git
# 设置跟踪关系
# 1.拉取并列出最新远程分支
git fetch --all #拉取所有最新远程分支，不会合并或修改本地代码
git branch -r #列出所有远程分支
# 2.追踪远程分支，简化pull、push操作，无需指定远程分支
git branch -vv #查看本地分支及其跟踪关系
git branch -u 远程仓库名/分支名 #设置本地分支与远程分支的追踪关系
```

```git
# 查看所有远程仓库
git remote -v
# 或使用详细模式
git remote show 别名
# 修改远程仓库别名
git remote 旧别名 新别名
# 添加一个新的远程仓库
git remote add 别名 仓库地址
# 修改现有远程仓库的URL
git remote set-url 别名 仓库地址
```

### git commit规范的标签
+ **feat**：提交新功能
+ **fix**：修复bug
+ **docs**：修改文档
+ **style**：调整代码格式，未修改代码逻辑（比如修改空格、格式化、缺少分号等）
+ **refactor**：代码重构，既没修复bug也没有添加新功能
+ **perf**：性能优化，提高性能的代码更改
+ **test**：添加或修改代码测试
+ **chore**：对构建流程或辅助工具和依赖库（如文档生成等）的更改

## Git其他常用命令
```git

git reset --hard commit-hash # 强制将本地分支重置到远程分支的某次提交
```

```git
git config --global pull.rebase true
```

## Git分支操作
在版本控制中，同时推进多个任务，我们就可以为每个任务创建一个分支。使用分支就可以把开发人员从开发主线上分离出来，在开发自己的分支的同时，不影响主线分支的运行。

分支合并冲突：同一个文件，被多个分支修改，当合并分支时，Git就不知道该如何合并文件内容（不知道谁去谁留）

需要手动解决冲突：在GitBash中编辑冲突文件，出现以下内容

```git
<<<<<<< HEAD			//当前分支标识符
hello git : hello world	//冲突内容
hello BugOS-ly			//冲突内容
=======					//文件冲突内容的分界线标识符
hello git				//冲突内容
hello BugOS-ly			//冲突内容
hot-fix:hello word		//冲突内容
>>>>>>> hot-fix			//被合并分支标识符

//解决冲突：直接删除冲突处不想保留的内容，记得删除"标识符"
hello git : hello world	//冲突内容
hello BugOS-ly			//冲突内容
hot-fix:hello word		//冲突内容
```

人为修改文件后，将该文件**git add 文件、git commit -m "日志信息"**；注意在**commit后面不能加文件名**，会报错，因为有多个分支修改了文件，Git不知道是提交哪个文件。

## Git团队协作机制
### 1、团队内协作
在GitHub上管理访问，添加collaborator(合作者)，被添加的collaborator就能够push代码到host的远程库

### 2、跨团队协作
外人将代码fork一份，外人可以修改代码，外人利用**"Pull request"**向原开发者发送**"拉取请求"**；原开发者会受到拉去请求，可以选择是否合并拉去请求。
